# i18n

Die eingebauten Komponenten-Texte des Kits (Platzhalter, ARIA-Labels, Buttontext) werden über einen einzigen Context-Provider lokalisiert.

## Standard

Ohne Provider verwenden alle Komponenten die **russische** Locale (`ru`). Texte wie `Очистить`, `Поиск...`, `Готово` kommen daher.

## Auf Englisch wechseln

Umschließe deine App mit `UidLocaleProvider` und der eingebauten `en`-Locale:

```vue
<script setup lang="ts">
import { UidLocaleProvider, en } from '@dskripchenko/ui'
</script>

<template>
  <UidLocaleProvider :locale="en">
    <App />
  </UidLocaleProvider>
</template>
```

Das war's. Alle eingebauten Texte (Clear-Button, Suchplatzhalter, Tour-Navigation usw.) werden Englisch.

## Eingebaute Locales

```ts
import { ru, en } from '@dskripchenko/ui'
```

- `ru` — Standard. Russisch.
- `en` — Englisch.

Beide implementieren das vollständige `UidLocale`-Interface.

## Partielle Überschreibungen

Du musst keine vollständige Locale liefern. Der Provider mergt deine Überschreibung tief auf das Default `ru`:

```vue
<UidLocaleProvider :locale="{ tour: { next: 'Vor', prev: 'Zurück' } }">
  <App />
</UidLocaleProvider>
```

Nicht angegebene Schlüssel behalten ihre `ru`-Werte. Gleiches gilt für Englisch:

```vue
<script setup lang="ts">
import { en } from '@dskripchenko/ui'
const locale = { ...en, tour: { ...en.tour, finish: 'Fertig!' } }
</script>

<template>
  <UidLocaleProvider :locale="locale">
    <App />
  </UidLocaleProvider>
</template>
```

## Eigene Locale

Liefere eine vollständige Locale via `UidLocale`:

```ts
import type { UidLocale } from '@dskripchenko/ui'

export const de: UidLocale = {
  common: {
    clear: 'Löschen',
    close: 'Schließen',
    search: 'Suchen...',
    confirm: 'Bestätigen',
    cancel: 'Abbrechen',
    loading: 'Wird geladen...',
    noResults: 'Keine Ergebnisse',
  },
  copy: { copy: 'Kopieren', copied: 'Kopiert' },
  select: { placeholder: 'Auswählen...', noResults: 'Keine Ergebnisse' },
  combobox: {
    placeholder: 'Tippen Sie...',
    noResults: 'Keine Ergebnisse',
    create: (label) => `+ "${label}" erstellen`,
  },
  datePicker: {
    placeholder: 'Datum wählen',
    months: ['Januar', 'Februar', 'März', 'April', 'Mai', 'Juni',
             'Juli', 'August', 'September', 'Oktober', 'November', 'Dezember'],
    weekdaysShort: ['Mo', 'Di', 'Mi', 'Do', 'Fr', 'Sa', 'So'],
    prevMonth: 'Vorheriger Monat',
    nextMonth: 'Nächster Monat',
  },
  // … siehe UidLocale-Typ für die vollständige Form
}
```

Verwendung wie bei den eingebauten:

```vue
<UidLocaleProvider :locale="de">
  <App />
</UidLocaleProvider>
```

## Locale im eigenen Code lesen

Wenn du eine Komponente auf dem Kit aufbaust und die aktive Locale verwenden willst:

```vue
<script setup lang="ts">
import { useLocale } from '@dskripchenko/ui'

const locale = useLocale()
</script>

<template>
  <span>{{ locale.common.clear }}</span>
</template>
```

`useLocale()` liefert eine `ComputedRef<UidLocale>`, die im Template automatisch entpackt wird.

## Programmatisches Provide

Wenn du die Locale programmatisch liefern willst (z. B. Sprachumschaltung per Dropdown):

```vue
<script setup lang="ts">
import { ref, computed } from 'vue'
import { provideLocale, ru, en } from '@dskripchenko/ui'

const lang = ref<'ru' | 'en'>('en')
provideLocale(computed(() => (lang.value === 'ru' ? ru : en)))
</script>

<template>
  <select v-model="lang">
    <option value="ru">Russisch</option>
    <option value="en">Englisch</option>
  </select>
  <App />
</template>
```

## Abdeckung

Alle Komponenten-Texte (Platzhalter, ARIA-Labels, Buttontext) werden über den Provider lokalisiert. Abgedeckte Komponenten:

Select, Combobox, DatePicker, DateRangePicker, TimePicker, TreeSelect, NumberInput, TagsInput, FileUpload, Mention, BackTop, Tour, TreeView, Pagination, Modal, Drawer, Toast, Alert, Tag, Code, DescriptionList, ColorPicker, Cascader, Calendar u. a.

## Per-Prop-Überschreibungen

Prop-Anpassungen haben weiterhin Vorrang vor der Locale. Beispiel `UidSelect`:

```vue
<UidSelect placeholder="Wählen Sie eine von drei..." :options="options" />
```

Der `placeholder`-Prop schlägt `locale.select.placeholder`. Die Locale greift nur, wenn der Prop weggelassen wird.

## Locale-Form

```ts
interface UidLocale {
  common: {
    clear: string; close: string; search: string
    confirm: string; cancel: string; loading: string; noResults: string
  }
  copy: { copy: string; copied: string }
  select: { placeholder: string; noResults: string }
  combobox: { placeholder: string; noResults: string; create: (label: string) => string }
  datePicker: {
    placeholder: string
    months: readonly string[]
    weekdaysShort: readonly string[]
    prevMonth: string; nextMonth: string
  }
  dateRangePicker: { placeholder: string; presetLast: (days: number) => string }
  timePicker: { placeholder: string; now: string; confirm: string }
  colorPicker: { placeholder: string }
  numberInput: { increment: (step: number | string) => string; decrement: (step: number | string) => string }
  tagsInput: { remove: (label: string) => string }
  treeView: { expand: string; collapse: string }
  treeSelect: { placeholder: string; removeTag: (label: string) => string }
  fileUpload: { primaryText: string; secondaryText: string; remove: (name: string) => string }
  backTop: { label: string }
  tour: { next: string; prev: string; finish: string; skip: string }
  rating: { label: (current: number, max: number) => string }
  mention: { noResults: string }
  modal: { close: string }
  drawer: { close: string }
  toast: { close: string }
  pagination: {
    first: string; prev: string; next: string; last: string
    page: (n: number) => string
    pageSize: (n: number) => string
  }
}
```

Für partielle Überschreibungen verwende `UidPartialLocale` — jeder verschachtelte Schlüssel wird optional.
